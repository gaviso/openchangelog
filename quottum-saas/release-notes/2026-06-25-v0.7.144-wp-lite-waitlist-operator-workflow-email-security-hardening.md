## Operator console — WP Lite waitlist becomes a real workflow

The previous read-only list at **/admin/wp-lite-waitlist** is now a full operator console.

### Detail page
Click any name in the list to land on `/admin/wp-lite-waitlist/[id]` — every captured field shown untruncated (current tool, top wish, IP, referrer host, user-agent, decided-by + decided-at).

### Actions (with confirm modals where state-changing)
- **Pending** → **Approve** *(fires "you're in" email with the download link + optional inline **cohort tag** for wave management)*, **Decline** *(silent — no applicant email)*, **Mark spam** *(hidden from default list, kept in DB for pattern detection)*
- **Approved** → **Resend approval email** *(handles "I never got it" tickets)*, **Reset to pending**
- **Declined / Spam** → **Reset to pending** *(misclick escape hatch)*

### Enhanced list
- **Status tabs** with live counts (Pending · Approved · Declined · Spam · All) — default = Pending so the working queue isn't drowned
- **Search** across name / email / current_tool / top_wish / cohort
- **Filter chips** for role + urgency
- **Bulk select + bulk approve/decline/spam** with optional cohort tag applied — "approve a wave of 50" in one click
- Each row's status chip + cohort chip visible at a glance

### Operator notes
Per-applicant **internal note** and **cohort tag** editable on the detail page. Separate PATCH endpoint so saving a note can never accidentally trigger a status change or applicant email.

### Audit trail
Every action writes to `admin_audit` (`wp_lite_waitlist.approve`, `.decline`, `.spam`, `.reset`, `.note_update`, `.resend_approval`) — full who-did-what-when from day one.

### Other
- Operator notification email's *"Open waitlist console"* button now **deep-links to the specific row** instead of the list.
- CSV export adds `status`, `decided_at`, `internal_note`, `cohort` to the exported columns.
- New approval email template `platform.wp_lite_waitlist_approved` — content-only, master design wraps at send time.

## Security hardening — applies to ALL outbound emails

Discovered + fixed during this work, but the fix is platform-wide:

- **Email header injection** — `{{var}}` substitution in subject lines now strips CR/LF, preventing a malicious form input from injecting `Bcc:` / `Subject:` SMTP headers.
- **HTML injection in email bodies** — `{{var}}` substitution now HTML-escapes values by default. Existing operator-authored template bodies still render normally; URL vars with `&` still parse correctly in `href`.
- **CSV formula injection** — the WP Lite waitlist CSV export now prefixes `'` to any cell value starting with `=`, `+`, `-`, `@`, or tab/CR, so a malicious applicant name can't run a formula when an operator opens the export in Excel/Numbers/LibreOffice.
