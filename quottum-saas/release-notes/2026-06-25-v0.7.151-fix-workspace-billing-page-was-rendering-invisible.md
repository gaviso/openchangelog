## Fix: workspace billing page was rendering invisible

`/dashboard/settings/billing` was wrapping its content in `<div className="quottum-set-panel">`. That class is defined as `display: none` by default — it's a tab-panel pattern that only becomes visible when `.active` is also applied. My billing page is the entire child of the settings layout (not a tab), so the wrapper hid the whole card on every workspace.

Caught by a workspace owner reporting the page rendered empty. Removed the wrapper.

The Phase 3 adversarial review flagged this class as a typo (renamed from `quottum-settings-panel` to `quottum-set-panel`), but the renamed class is also hidden by default — the review didn't catch the `display: none` rule. Other working settings pages don't use the class at all.

After this release, the v0.7.150 beta-plan card (and the paid plan card, and the Free-plan placeholder) all render correctly on the billing page.
