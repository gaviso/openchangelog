Focused signup-flow fixes triggered by smoke-testing v0.7.181's terminate path.

### Fixed

- **False reconsent prompt for new signups.** Users coming through the email-confirmation wall and creating their workspace via \`/onboarding\` were landing on \`/dashboard\` with zero \`tos_acceptances\` rows — the reconsent gate falsely demanded they accept the TOS they'd just clicked through minutes ago. \`/api/auth/signup-workspace\` now records one \`tos\` + one \`privacy_ack\` row at first-workspace creation, with the captured IP/UA and \`version=TOS_VERSION\`. Instant-signup path remains unchanged (it already POSTs \`/api/tos/accept\` directly); duplicate rows are harmless since the gate reads the latest per document.
- **Onboarding "You picked" callout** — swapped the v0.7.178 holdover \`green-50/100/700\` + \`text-gray-*\` tokens for theme-aware \`accent-primary-light\`, \`accent-primary-border\`, \`success\`, and \`ink-*\`. Same fix as v0.7.179 SignupForm.
- **ReconsentGate effective date** renders as \`"June 27, 2026"\` (US long-form) instead of the raw \`"2026-06-27"\` constant. Parses YYYY-MM-DD parts explicitly so the day doesn't shift in negative-UTC browsers.

### Added

- **\`SignupSuccessCard\`** — dedicated post-signup card that replaces the entire signup form once \`auth.signUp()\` succeeds and email confirmation is required. Previous behavior stuffed the success message into the red error banner and left the form mounted (email + password still populated, "Start trial" CTA that would re-submit). New card carries: green-check icon, "Check your inbox at <email>", plan/trial context tailored to free vs trial vs paid-no-trial vs invite, a **Resend confirmation** button (\`supabase.auth.resend\`), an "Already confirmed? Sign in" link, and a "Wrong email? Start over" button that clears state + password.

### Existing-user note

Users with workspaces created before this release have a (potentially missing) \`tos_acceptances\` row. The new code path is first-workspace-only, so it doesn't backfill them. If you've never accepted the current TOS_VERSION, you'll see the reconsent modal once on next dashboard load and clicking through writes the row.
