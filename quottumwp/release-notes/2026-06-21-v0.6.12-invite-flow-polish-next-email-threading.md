The invite-acceptance flow finally works end-to-end for users who don't already have a Quottum account. Three small fixes that compound into a much smoother experience.

### What broke

Clicking an invite link with no session would bounce you to /login. You'd type the invited email, get \"Invalid login credentials\" (because no account exists yet), click \"Create one\", sign up — and land on /dashboard with a brand-new personal \"John's workspace\" instead of being added to the workspace you were invited to. Total breadcrumb loss.

### What's fixed

**\`/login\`**
- Honors \`?next=\` and \`?email=\` query params
- Email pre-filled when supplied — no retyping
- After sign-in, hard-navigates to the \`next\` destination instead of always /dashboard
- \"Create one\" link carries \`next\` + \`email\` forward

**\`/signup\`**
- Same \`?next=\` + \`?email=\` honoring
- When \`next\` starts with \`/invite/\`, **skips the personal-workspace creation step** — you're being added to an existing workspace, no point cluttering the switcher
- Green banner: \"Continuing from a workspace invitation — you'll be added to the workspace right after signup.\"
- After sign-up, hard-navigates to \`next\` (lands back on the invite page)
- \"Sign in\" link carries everything back

**\`/invite/[token]\`**
- When the accept POST returns 401, the redirect to /login now passes \`?email=\` so the user sees which account they need
- The existing \"Sign up\" link also gets \`?email=\` pre-populated

**Open-redirect guard** — \`safeNext()\` in both auth pages only honors paths starting with a single \`/\` (no \`//\` or full URLs). Pasting \`?next=https://evil.com\` falls back to /dashboard.

### End-to-end now

1. Click invite link, no session → \`/login?next=/invite/X&email=admin@gaviso.agency\`
2. \"No account? Create one\" → \`/signup?next=/invite/X&email=admin@gaviso.agency\` (banner shown, email pre-filled)
3. Set password → land back on \`/invite/X\` automatically
4. Click \"Accept invitation\" → in
