You can finally rename a workspace. Until now the name was set once at signup (\"{First name}'s workspace\") or creation and permanently stuck — there was no UI or API to change it.

### What's new

**Settings → General** (new first tab; the settings index now lands here):

- **Workspace name** — editable by owners. Non-owner members see it disabled with a note.
- **Workspace ID (slug)** — shown read-only. It's a stable identifier and isn't regenerated on rename, so nothing breaks.
- Copy spells out the distinction that prompted this: this is your **internal** name (workspace switcher), separate from the client-facing **Company name** on the Branding tab.

After saving, the sidebar switcher and topbar update immediately.

### API

New \`PATCH /api/workspaces/[id] { name }\`, ownership-gated:
- Not a member → 403
- Member but not an owner → 403 with \"Only workspace owners can rename the workspace.\"
- Name-only (slug is immutable), 80-char cap

### Workspace name vs Company name — the difference

| | Set where | Seen by |
|---|---|---|
| **Workspace name** | Settings → General | You + your team (switcher) |
| **Company name** | Settings → Branding | Your clients (quotes, contracts, PDFs) |

They're usually the same for a single business, but can differ — e.g. an agency running client brands as separate workspaces.
