The clients module gets the v0.3 contracts treatment: stage taxonomy, owner attribution, sortable register, full-profile detail page. Plus a brand-new **public client portal** at /p/[token] that gives each client a single page listing every quote and contract they've ever received.

### Clients

- **Stage taxonomy**: \`lead\` / \`active\` / \`paused\` / \`lost\`. Filter tabs on the list with live counts.
- **Owner column** (\`owner_user_id\`) — \"You\" badge for clients you own. Multi-user-friendly out of the gate.
- **Sortable register** with text search across company / contact / email.
- **Detail rebuild** — combined profile with header chips, KPI strip (TCV / MRR / open quotes / accepted), quotes + contracts sections, contact strip with inline icons.

### ClientPicker combobox

Replaces the native \`<select>\` in QuoteBuilder + ContractForm with a search-as-you-type picker — typeahead, keyboard nav (↑/↓/Enter/Esc), and contact preview under each option. Foundation for autofilling line-item email/phone in v0.5.0.

### Token rotation + expiry

- New \`POST /api/tokens/rotate { kind, id, expires_in_days? }\` — generates a fresh token, optionally setting an expiry. **Old token is overwritten — no grace window** (rotation IS revocation).
- Public quote pages honor \`token_expires_at\` (expired = 404, no status leakage).
- Same \`token_expires_at\` column added to clients + quotes + contracts so future rotation flows have a single field to set.

### Public client portal — **/p/[token]**

Each client can have a unique link that gives them one page with every quote and contract you've issued them:

- **Quote rows link straight to /q/[token]** for the full interactive document.
- **Contract rows display read-only** for now (the full public contract page lands in v0.5.0).
- Honors expiry — links whose own token is missing or expired render as muted.

Workspace branding shows in the header strip.

### Schema (migration 010)

```
clients:
  + owner_user_id    uuid references auth.users
  + stage            text default 'active'
  + client_token     text unique
  + token_expires_at timestamptz
quotes:
  + token_expires_at timestamptz
contracts:
  + token_expires_at timestamptz
+ RLS: public can view client by token
+ indexes on owner_user_id, stage, client_token
```

### Try it

- **/dashboard/clients** — filter tabs, search, sort
- **/dashboard/clients/[id]** — rebuilt detail page on Apex / Cloud Nine / Halo / etc.
- **/p/apex-portal-demo-...** — copy the token from \`select client_token from clients\` and try the portal
- **Quote builder / Contract form** — try the new client combobox
