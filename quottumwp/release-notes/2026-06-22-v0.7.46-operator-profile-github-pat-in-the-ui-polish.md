This release is **operator-internal** — improvements to the Quottum admin console that don't change anything customers see in their workspaces. No action required from workspace owners or members.

## 👤 Operator profile page

A new `/admin/profile` is now the single hub for an operator's account.

- **Edit your details** — change your full name, change your email (with the standard email-change confirmation), upload a profile photo.
- **Two-factor authentication** is now managed from here too (the standalone Security page redirects in).
- **Appearance** (System / Light / Dark) — moved out of the sidebar into the profile. The setting is shared with the dashboard.

## 🔌 GitHub integration — token in the UI

Connect Quottum to its private GitHub repo without ever editing env vars.

- Paste a fine-grained Personal Access Token directly in the operator console (`/admin/changelog`); rotate by pasting a new one. Tokens are write-only — the UI only ever shows the last four characters.
- **Test connection** button does a real probe and reports exactly what GitHub returns (`200` / `401` / `403` / `404`) so misconfigurations are obvious instead of silent.
- **Token-expiry awareness:** the token's expiration date is captured automatically. A banner appears across `/admin/*` when ≤14 days remain (amber) or after it's lapsed (red), and platform admins get an email at the 14d → 7d → 3d → 1d → expired thresholds.

This is the foundation for the upcoming admin release-timeline view.

## ✨ Operator-console polish

- **Feature Flags toggles** redesigned: clearly visible off-state, accent dot for "overridden" instead of an outer ring (no more "is that on or off?"), reset arrow paired with the dot, knob no longer hangs off the track.
- Same off-state contrast fix applied to the attachment **client-visible** switch.

## Behind the scenes (no schema impact for workspaces)

- New tables: `admin_github_settings` (singleton, RLS-locked), seeded `platform.github_token_expiring` email template.
- `/api/me/profile` + `/api/me/avatar` endpoints (avatar served via auth-gated 302 to a fresh presigned B2 URL).
- Admin layout fires a throttled (1×/hour) check on every render that emails platform admins once per warning band as the GitHub token nears expiry.
