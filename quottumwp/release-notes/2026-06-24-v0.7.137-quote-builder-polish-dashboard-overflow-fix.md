## Quote builder polish

- **Explicit Apply + Remove** on the Discount card. Type a value, click **Apply** to commit it; the status line confirms *"✓ Applied: −$500 off subtotal"* with a **Remove** action next to it. Matches the cadence of the Coupon tab.
- **Coupon Apply now actually works.** The button was previously disabled with a stale comment; it now POSTs to a new `/api/coupons/preview` and shows *"✓ Saves $750 on this quote"* or the validation reason inline, in real time. Uses the same `validateAndPriceCoupon` as the save path so preview and commit can't disagree.
- **Primary CTA renamed** `Create quote` → **Publish**, matching the internal status it actually sets and pairing cleanly with `Save draft`.

## Dashboard polish

- **Contract register no longer clips dates.** Switched the table to a fixed layout with column widths sized to fit inside the card; long titles and client names truncate with a hover tooltip instead of pushing the rest of the columns out of view. Dropped the redundant Status column — the At-risk renewals rail right next to it already surfaces the same info.
